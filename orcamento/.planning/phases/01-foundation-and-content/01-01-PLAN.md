# Plan 01-01: Project Setup

## Goal

Establish the project scaffold — directory structure, Tailwind CSS v4 build pipeline, vercel.json config, and a skeleton index.html — then perform the first Vercel deploy to obtain the production URL. This URL is required by plan 01-02 to write the absolute og:image href.

## Requirements

TECH-01, TECH-02, TECH-04

## must_haves

- `npm install` succeeds with `tailwindcss@4.2.1` and `@tailwindcss/cli@4.2.1` as devDependencies
- `npx @tailwindcss/cli -i src/input.css -o dist/output.css --minify` runs without error and produces a non-empty `dist/output.css`
- `vercel.json` exists at root with `cleanUrls: true` and two cache-header rules (one for `/*`, one for `/assets/*`)
- `index.html` exists at root with `<!DOCTYPE html>`, `<html lang="pt-BR">`, a complete `<head>` (charset, viewport, title, font preconnect + stylesheet link to `dist/output.css`), an empty `<body>`, and OG tags with placeholder values
- Project is pushed to GitHub and auto-deployed to Vercel; the deployed URL is reachable (HTTP 200)
- `dist/output.css` is committed to git (not gitignored) so Vercel can serve it without a build command, OR Vercel is configured with build command `npx @tailwindcss/cli -i src/input.css -o dist/output.css --minify` and install command `npm install`

## Context

Stack: HTML5 + Tailwind CSS v4 (via CLI) + Inter variable font via Google Fonts CDN. No JavaScript in Phase 1. No `tailwind.config.js` — Tailwind v4 uses CSS-first config via `@theme` in `input.css`.

Vercel deploy strategy: configure Vercel with build command and install command so `dist/output.css` is built fresh on each deploy (cleaner than committing the artifact). If Vercel build command approach is chosen, do NOT add `dist/output.css` to `.gitignore` until Vercel is confirmed building it.

The production URL format will be `https://orcamento-brum.vercel.app` — use this as the base for OG tags in plan 01-02 after confirming the deploy succeeds.

<task id="01-01-T1">
<title>Initialize npm project and install Tailwind CSS v4</title>
<files>package.json, src/input.css</files>
<instructions>
Run `npm init -y` in the project root to create package.json.

Install Tailwind CSS v4 and its CLI:
```
npm install -D tailwindcss@4.2.1 @tailwindcss/cli@4.2.1
```

Create `src/input.css` with exactly this content:
```css
@import "tailwindcss";

@theme {
  --color-brand-primary: #0f172a;
  --color-brand-accent: #16a34a;
  --color-brand-muted: #64748b;
  --font-sans: 'Inter', sans-serif;
}
```

Verify the CLI works by running the production build command:
```
npx @tailwindcss/cli -i src/input.css -o dist/output.css --minify
```

Confirm `dist/output.css` is created and non-empty (file size > 0 bytes).

Do NOT create `tailwind.config.js` — this is a Tailwind v4 project. The `@theme` block in `input.css` is the only config needed.
</instructions>
<verify>
```bash
node -e "const p=require('./package.json'); const v=p.devDependencies; console.assert(v['tailwindcss']==='4.2.1'); console.assert(v['@tailwindcss/cli']==='4.2.1'); console.log('deps ok')"
ls -s dist/output.css | awk '{if($1>0) print "output.css non-empty"; else exit 1}'
grep -q '@import "tailwindcss"' src/input.css && echo "input.css ok"
test ! -f tailwind.config.js && echo "no config.js"
```
</verify>
<done>package.json has tailwindcss@4.2.1 and @tailwindcss/cli@4.2.1 as devDependencies; dist/output.css exists and is non-empty; src/input.css contains @import "tailwindcss" and the @theme block; no tailwind.config.js exists.</done>
</task>

<task id="01-01-T2">
<title>Create vercel.json and skeleton index.html</title>
<files>vercel.json, index.html, .gitignore</files>
<instructions>
Create `vercel.json` at the project root:
```json
{
  "$schema": "https://openapi.vercel.sh/vercel.json",
  "cleanUrls": true,
  "headers": [
    {
      "source": "/(.*)",
      "headers": [
        { "key": "Cache-Control", "value": "public, max-age=3600, stale-while-revalidate=86400" }
      ]
    },
    {
      "source": "/assets/(.*)",
      "headers": [
        { "key": "Cache-Control", "value": "public, max-age=31536000, immutable" }
      ]
    }
  ]
}
```

Create `index.html` at the project root with this skeleton:
```html
<!DOCTYPE html>
<html lang="pt-BR">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Proposta Comercial — MVP Delivery de Chopp | Marcus Gonçalves</title>
  <meta name="description" content="Sistema completo de pedidos, automação WhatsApp e painel de gestão. 45 dias. R$ 12.000.">
  <meta property="og:title" content="Proposta Comercial — MVP Delivery de Chopp">
  <meta property="og:description" content="Sistema completo de pedidos, automação WhatsApp e painel de gestão. 45 dias. R$ 12.000.">
  <meta property="og:image" content="PLACEHOLDER_REPLACE_AFTER_DEPLOY">
  <meta property="og:url" content="PLACEHOLDER_REPLACE_AFTER_DEPLOY">
  <meta property="og:type" content="website">
  <link rel="preconnect" href="https://fonts.googleapis.com">
  <link rel="preconnect" href="https://fonts.gstatic.com" crossorigin>
  <link href="https://fonts.googleapis.com/css2?family=Inter:ital,opsz,wght@0,14..32,100..900;1,14..32,100..900&display=swap" rel="stylesheet">
  <link rel="stylesheet" href="dist/output.css">
</head>
<body class="font-sans bg-white text-slate-900 antialiased">
  <p class="p-8 text-slate-500">Proposta em construção.</p>
</body>
</html>
```

Create `assets/img/` directory (create a `.gitkeep` placeholder so git tracks the directory):
```
mkdir -p assets/img
touch assets/img/.gitkeep
```

Create `.gitignore`:
```
node_modules/
```

Do NOT add `dist/` to `.gitignore`. The compiled CSS must either be committed or built by Vercel. Commit `dist/output.css` to git for now — it ensures the deploy works regardless of Vercel build config.
</instructions>
<verify>
```bash
python3 -c "import json; d=json.load(open('vercel.json')); assert d['cleanUrls']==True; assert len(d['headers'])==2; print('vercel.json ok')"
grep -q 'lang="pt-BR"' index.html && echo "html lang ok"
grep -q 'charset="UTF-8"' index.html && echo "charset ok"
grep -q 'dist/output.css' index.html && echo "stylesheet link ok"
grep -q 'node_modules/' .gitignore && echo ".gitignore ok"
test -f assets/img/.gitkeep && echo "assets dir ok"
```
</verify>
<done>vercel.json exists with cleanUrls:true and two headers entries; index.html exists with lang="pt-BR", charset, viewport, title, OG placeholder tags, and stylesheet link; .gitignore excludes node_modules; assets/img/.gitkeep exists.</done>
</task>
