# Plan 01-04: CSS Mobile-First — Design System e Estilização Completa

## Goal

Apply the complete mobile-first visual design to every section in `index.html` using Tailwind CSS v4 utility classes. After this plan, the page is shippable: professional typography, clear visual hierarchy, readable on a 375px screen without horizontal scroll, and all CTAs tappable at minimum 48px height.

## Requirements

CONV-02, CONV-03, TECH-04

(All CONT-* requirements are fulfilled by plan 01-03; this plan fulfills the presentation layer that makes the content legible and professional.)

## must_haves

- Page renders without horizontal scroll at 375px viewport width — zero overflow-x on any element
- All body text is at minimum 16px (Tailwind `text-base` or larger)
- Both WhatsApp CTA buttons are at minimum 48px tall (Tailwind `min-h-[48px]` or `py-4` which yields ~56px)
- Inter font renders via Google Fonts CDN — `font-family` on `body` is Inter, not a system font fallback
- Visual section order matches HTML order — no CSS positioning that reorders content visually
- At least 64px vertical gap between each major section (Tailwind `py-16` = 64px or greater)
- CTA buttons use green background (`bg-green-600`) to signal WhatsApp association
- Investment value "R$ 12.000" is visually prominent — larger text, distinct color or weight
- Page is legible on both 375px mobile and 1024px desktop without requiring any JavaScript
- `dist/output.css` is rebuilt with the final Tailwind classes — file must be re-generated after adding classes to HTML

## Context

Design constraints (locked):
- Font: Inter variable (already loaded via Google Fonts CDN in `<head>`)
- Mobile-first: ALL base Tailwind classes target 375px. `md:` prefix (768px+) for enhancement only. Never write desktop-first and override for mobile.
- Color palette (defined in `src/input.css` @theme): brand-primary #0f172a, brand-accent #16a34a, brand-muted #64748b, surface #f8fafc
- No JavaScript: Phase 1 has zero JS. Every interactive state (hover, focus) is pure CSS.

Section background alternation (for visual rhythm):
- `#hero`: dark background (`bg-slate-900` text-white) — high-impact first impression
- `#problema`: white (`bg-white`)
- `#solucao`: light surface (`bg-slate-50`)
- `#escopo`: white (`bg-white`)
- `#timeline`: light surface (`bg-slate-50`)
- `#investimento`: dark background (`bg-slate-900` text-white) — emphasizes the price reveal moment
- `#pagamento`: white (`bg-white`)
- `#diferenciais`: light surface (`bg-slate-50`)
- `#sobre`: white (`bg-white`)
- `#cta`: green accent (`bg-green-600` text-white) — conversion moment
- `footer`: dark (`bg-slate-900` text-slate-400)

Tailwind v4 note: the CLI is `npx @tailwindcss/cli`, not `npx tailwindcss`. After adding classes, rebuild with:
```
npx @tailwindcss/cli -i src/input.css -o dist/output.css --minify
```

Anti-patterns to avoid:
- `max-w-*` without `mx-auto px-6` — content will touch screen edges on mobile
- `text-sm` on body copy — minimum is `text-base` (16px)
- Fixed heights on containers — use `min-h-*` or let content determine height
- `overflow-hidden` on the `<body>` — can mask horizontal scroll bugs; test with DevTools
- Margin-only spacing between sections — use section padding (`py-16`) for consistent spacing regardless of adjacent elements
